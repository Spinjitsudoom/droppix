# Stylus Support Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Forward an Android tablet's stylus to the host as a real graphics-tablet pen (pressure + eraser end), so drawing apps get pressure and the eraser — via a new `Pen` message and a new host uinput pen device bound to the droppix output.

**Architecture:** Mirrors the existing input model (client normalizes → host replays). Android detects a stylus in `onTouchEvent` (`getToolType`) and sends a `Pen` message instead of finger `Touch`; the host has a separate uinput pen device (`BTN_TOOL_PEN`/`BTN_TOOL_RUBBER`/`ABS_PRESSURE`) bound to the droppix output. **Android-only; pressure + eraser (no tilt/hover). No HELLO/version change.**

**Tech Stack:** C++ (host, Qt6 GUI n/a), Kotlin/Android, CMake + Gradle.

## Global Constraints

- **No HELLO/version bump.** `Pen` is an independent control message like Touch/Scroll/Key. Do NOT touch `kProtocolVersion`/`Protocol.VERSION`/HELLO.
- **Message numbering:** append `Pen = 15` after `Key = 14` (do NOT renumber existing).
- **Wire (C++ ↔ Kotlin identical):** `Pen` body = `u16 x, u16 y, u16 pressure, u8 flags` (big-endian). `x/y` 0..65535 (droppix monitor, same normalization as touch); `pressure` 0..1023; `flags` bit0 = touching (contact), bit1 = eraser.
- **Host pen device:** separate uinput device (`<touch_name>-pen`), `INPUT_PROP_DIRECT`, `ABS_X/Y` 0..65535 (bound to the droppix output so it spans that monitor), caps `BTN_TOOL_PEN`/`BTN_TOOL_RUBBER`/`BTN_TOUCH`/`ABS_PRESSURE`. Distinct vendor/product id. Non-fatal on failure.
- **Android:** a stylus/eraser pointer produces `Pen` events ONLY (never also finger `Touch`); fingers unchanged.
- **Gating:** pen injection wired only when the injector is active (the existing `cfg_.touch` block); handler reset to `nullptr` at session start with the others.
- **Build/test envs** (repo on CIFS no-exec mount):
  - Host: `distrobox enter droppix-dev -- bash -lc 'cmake --build ~/droppix-build -j && ctest --test-dir ~/droppix-build <-R filter> --output-on-failure'` (configure once with `-DDROPPIX_BUILD_TESTS=ON`).

    ```cpp
    void InputInjector::pen(uint16_t x, uint16_t y, uint16_t pressure, bool touching, bool eraser) {
      if (pen_fd_ < 0) return;
      if (touching && !pen_down_) {              // tool enters proximity on the down edge
        pen_eraser_ = eraser;
        emit(pen_fd_, EV_KEY, eraser ? BTN_TOOL_RUBBER : BTN_TOOL_PEN, 1);
        pen_down_ = true;
      }
      emit(pen_fd_, EV_ABS, ABS_X, x);
      emit(pen_fd_, EV_ABS, ABS_Y, y);
      emit(pen_fd_, EV_ABS, ABS_PRESSURE, pressure);
      emit(pen_fd_, EV_KEY, BTN_TOUCH, touching ? 1 : 0);
      emit(pen_fd_, EV_SYN, SYN_REPORT, 0);
      if (!touching && pen_down_) {               // tool leaves proximity on the up edge
        emit(pen_fd_, EV_KEY, pen_eraser_ ? BTN_TOOL_RUBBER : BTN_TOOL_PEN, 0);
        emit(pen_fd_, EV_SYN, SYN_REPORT, 0);
        pen_down_ = false;
      }
    }
    ```

## Self-review notes

- **Spec coverage:** protocol (T1/T5), host pen device (T2), pen→output binding (T3), dispatch+daemon (T4), Android capture (T6), verify (T7). Every spec section maps to a task.
- **Wire consistency:** C++ `encode_pen` = `{x_hi,x_lo,y_hi,y_lo,p_hi,p_lo,flags}`; Kotlin `encodePen` identical; both tests assert the `40000/20000/900/0x03` byte layout.
- **No version bump:** `Pen = 15`/`PEN(15)` appended; HELLO untouched.
- **Type consistency:** `pen(uint16_t,uint16_t,uint16_t,bool,bool)`, `set_pen_handler(void(u16,u16,u16,u8))`, `sendPen(Int,Int,Int,Int)`, flags bit0=touching/bit1=eraser everywhere.
- **Known risk:** KWin pen→output binding (T3) — the pen is a tablet, not in `ListTouch`; the KWin path enumerates all devices best-effort and is validated on-device, with a `scale_x/scale_y` fallback if KWin can't bind a tablet to an output.
- **Fingers untouched:** the stylus branch is before the finger path and returns; finger `Touch` behavior is unchanged.
